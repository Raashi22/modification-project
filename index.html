Array Data Structure

Definition: An Array is a linear data structure that stores elements of the same data type in contiguous memory locations.

Key Concepts:

  • Index: Each element has a unique index (starting from 0)
  • Contiguous Memory: Elements are stored in adjacent memory locations
  • Fixed Size: Size is determined at declaration (in most languages)
  • Random Access: Can access any element directly using its index

Operations:

  • Insert at End: Add element at the end of the array
  • Insert at Index: Add element at a specific position
  • Delete at Index: Remove element from a specific position
  • Delete from End: Remove the last element
  • Search: Find the index of a specific value
  • Access: Get element at a specific index

Advantages:

  • Random access - O(1) time complexity
  • Cache-friendly due to contiguous memory
  • Simple implementation
  • Memory efficient (no extra pointers)

Disadvantages:

  • Fixed size (in most implementations)
  • Insertion/deletion at beginning/middle is expensive
  • Memory waste if array is not fully utilized
  • No dynamic resizing in basic arrays

Time Complexity:

  • Access: O(1)
  • Search: O(n)
  • Insert at End: O(1)
  • Insert at Index: O(n)
  • Delete at End: O(1)
  • Delete at Index: O(n)

Real-world Applications:

  • Image processing (pixel arrays)
  • Mathematical computations
  • Database records
  • Lookup tables
  • Implementation of other data structures

Stack Visualization

Stack Data Structure

Definition: A Stack is a linear data structure that follows the LIFO (Last In, First Out) principle.

Key Concepts:

  • LIFO: Last In, First Out - the last element added is the first one removed
  • Top: The element at the top of the stack (most recently added)
  • Bottom: The element at the bottom of the stack (first added)

Operations:

  • Push: Adds an element to the top of the stack
  • Pop: Removes and returns the top element from the stack
  • Peek: Returns the top element without removing it
  • isEmpty: Checks if the stack is empty

Real-world Applications:

  • Function call stack in programming
  • Undo operations in text editors
  • Browser back button
  • Expression evaluation

Time Complexity:

  • Push: O(1)
  • Pop: O(1)
  • Peek: O(1)

Queue Visualization

Queue Data Structure

Definition: A Queue is a linear data structure that follows the FIFO (First In, First Out) principle.

Key Concepts:

  • FIFO: First In, First Out - the first element added is the first one removed
  • Front: The element at the front of the queue (first to be removed)
  • Rear: The element at the rear of the queue (most recently added)

Operations:

  • Enqueue: Adds an element to the rear of the queue
  • Dequeue: Removes and returns the front element from the queue
  • Front: Returns the front element without removing it
  • isEmpty: Checks if the queue is empty

Real-world Applications:

  • Print job scheduling
  • CPU task scheduling
  • Breadth-First Search (BFS)
  • Customer service lines

Time Complexity:

  • Enqueue: O(1)
  • Dequeue: O(1)
  • Front: O(1)

Singly Linked List Visualization

Singly Linked List Data Structure

Definition: A Singly Linked List is a linear data structure where each element (node) contains data and a reference to the next node.

Key Concepts:

  • Node: Contains data and a pointer to the next node
  • Head: The first node in the linked list
  • Tail: The last node in the linked list (points to null)
  • Null pointer: Indicates the end of the list

Operations:

  • Insert at Start: Add a new node at the beginning
  • Insert at End: Add a new node at the end
  • Delete at Index: Remove a node at a specific position
  • Traversal: Visit each node in the list

Advantages:

  • Dynamic size - grows and shrinks as needed
  • Efficient insertion and deletion at the beginning
  • No wasted memory

Disadvantages:

  • No random access - must traverse from head
  • Extra memory for pointers
  • Cannot traverse backwards

Time Complexity:

  • Insert at Start: O(1)
  • Insert at End: O(n)
  • Delete at Index: O(n)
  • Search: O(n)

Doubly Linked List Visualization

Doubly Linked List Data Structure

Definition: A Doubly Linked List is a linear data structure where each node contains data and references to both the next and previous nodes.

Key Concepts:

  • Node: Contains data, next pointer, and previous pointer
  • Bidirectional: Can traverse both forward and backward
  • Head: The first node (prev = null)
  • Tail: The last node (next = null)

Advantages over Singly Linked List:

  • Bidirectional traversal
  • Easier deletion of nodes
  • Can access previous node without traversal

Disadvantages:

  • Extra memory for previous pointers
  • More complex implementation
  • More pointer updates during operations

Time Complexity:

  • Insert at Start/End: O(1)
  • Delete at Start/End: O(1)
  • Insert/Delete at Index: O(n)
  • Search: O(n)

Circular Linked List Visualization

Circular Linked List Data Structure

Definition: A Circular Linked List is a linked list where the last node points back to the first node, forming a circle.

Key Concepts:

  • Circular Structure: Last node points to first node
  • No Null Pointers: Every node has a successor
  • Head Reference: Points to any node in the circle
  • Infinite Traversal: Can traverse indefinitely

Types:

  • Singly Circular: Each node points to next node
  • Doubly Circular: Each node points to both next and previous

Applications:

  • Round-robin scheduling
  • Music playlist (repeat mode)
  • Game player rotation
  • Circular buffer implementation

Advantages:

  • Efficient for round-robin operations
  • No need to check for end of list
  • Good for applications requiring cycling

Time Complexity:

  • Insert at Start/End: O(1)
  • Delete: O(n) in worst case
  • Search: O(n)

Binary Search Tree (BST) Data Structure

Definition: A Binary Search Tree is a hierarchical data structure where each node has at most two children, and the values follow a specific ordering property.

BST Properties:

  • Left Subtree: All values are less than the parent node
  • Right Subtree: All values are greater than the parent node
  • No Duplicates: Each value appears only once
  • Hierarchical: Tree-like structure with levels

Operations:

  • Insert: Add a new node while maintaining BST property
  • Delete: Remove a node and restructure the tree
  • Search: Find a value by traversing the tree
  • Traversal: Visit all nodes in a specific order

Traversal Types:

  • In-order: Left → Root → Right (gives sorted order)
  • Pre-order: Root → Left → Right (useful for copying trees)
  • Post-order: Left → Right → Root (useful for deletion)

Applications:

  • Database indexing
  • File system organization
  • Symbol tables in compilers
  • Priority queues

Time Complexity:

  • Search: O(log n) average, O(n) worst case
  • Insert: O(log n) average, O(n) worst case
  • Delete: O(log n) average, O(n) worst case

Graph Data Structure

Definition: A Graph is a non-linear data structure consisting of vertices (nodes) and edges that connect these vertices.

Key Concepts:

  • Vertex (Node): A point in the graph
  • Edge: A connection between two vertices
  • Adjacency: Two vertices connected by an edge
  • Degree: Number of edges connected to a vertex

Graph Types:

  • Undirected: Edges have no direction
  • Directed: Edges have direction (arrows)
  • Weighted: Edges have associated weights
  • Unweighted: Edges have no weights

Traversal Algorithms:

  • Depth-First Search (DFS): Explore as far as possible along each branch
  • Breadth-First Search (BFS): Explore all neighbors before moving to next level

Applications:

  • Social networks (friends connections)
  • Navigation systems (roads and intersections)
  • Computer networks
  • Dependency resolution
  • Game development (game maps)

Time Complexity:

  • Add Node: O(1)
  • Add Edge: O(1)
  • Remove Node: O(V + E)
  • DFS/BFS: O(V + E)
